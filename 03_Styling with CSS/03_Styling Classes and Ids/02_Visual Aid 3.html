Demo 2: When Order Matters (Same Specificity)
CSS Rules (in this order) .first-rule { background: yellow; }
.second-rule { background: lightblue; }
HTML <div class="first-rule second-rule">
<div class="second-rule first-rule">
Both classes, but second-rule CSS came last!
Order in HTML doesn't matter - still light blue!

Demo 3: Multiple Classes & Chaining
CSS Rules .bold { font-weight: bold; }
.italic { font-style: italic; }
.underline { text-decoration: underline; }
.bold.italic { color: purple; }
.bold.italic.underline { background: yellow; }
HTML Examples <span class="bold">
<span class="italic">
<span class="bold italic">
<span class="bold italic underline">
Just Bold Just Italic Bold + Italic = Purple! All Three = Yellow Background!

üèÜ Key Takeaways
  1. Specificity Hierarchy: ID (100) > Class (10) > Element (1)
  2. Order Matters: When specificity is equal, last rule wins
  3. Multiple Classes: Elements can have many classes, get all their styles
  4. Chaining: .class1.class2 targets elements with BOTH classes
  5. Grouping: Use commas to apply same styles to multiple selectors
